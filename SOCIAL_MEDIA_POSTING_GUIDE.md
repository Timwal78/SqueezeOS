# Social Media Posting Guide — 4-Step Cheat Code SOP

A systematic approach for maximizing content distribution and reach on algorithmic social platforms.

---

## Step 1: Anchor the First 3 Lines (4-Head Attention Model)

The shallow 2-layer model reads the beginning of your post to decide what it is about. Vague openers drop the distribution score.

**Rule:** Put your most heavy-hitting, industry-specific keywords in the first 15–30 words.

| Weak opener | Strong opener |
|---|---|
| "Hey guys, I wanted to talk about a cool project I've been working on lately..." | "Building a high-throughput data pipeline using an optimized Two-Tower architecture..." |

**Checklist:** Front-loaded? Keywords are in the first 2 lines.

---

## Step 2: Write Clean, Punchy Sentences (256-Dim Embeddings)

The system compresses complex text into simple categories using low-dimensional memory space. Nested or poetic phrasing causes the model to misclassify content.

**Rule:** Write in short, direct Subject-Verb-Object format. Bullet points force clean token clustering that the 256-dim model categorizes as "High-Value Technical Content."

- One idea per sentence
- Domain terminology over filler phrases
- Bullet lists over dense paragraphs

**Checklist:** Direct syntax? No fluff, clean phrasing, high density of domain terms.

---

## Step 3: Never Post Raw Outbound Links (Grox Filter Bypass)

The automated `grox/` engine flags platform leakage. An external link in the main body instantly tanks distribution weight or triggers spam filters.

**Rule:** Keep all value native to the platform.

- Place external destinations in your **profile bio**
- Use **native media elements** (images, video, carousels) to keep users inside the app ecosystem
- Never paste raw URLs in post body text

**Checklist:** No links? Kept 100% native.

---

## Step 4: Space Out Your Uploads (Defeat Author Decay)

The Home Mixer applies exponential decay if the same creator appears too many times in a user's feed back-to-back. Posting 3 times in one hour means you compete against yourself.

**Rule:** Space primary technical deep-dives by **at least several hours** between posts. This lets the candidate pool refresh so new posts are not suppressed by author diversity caps.

**Checklist:** Timed right? At least a few hours since the last post.
